# Principles

Use these references as inspiration when shaping a daily plan. Do not over-explain them unless the user asks.

## Useful Lenses

- Stephen Covey / Eisenhower: distinguish important work from merely urgent work.
- Ivy Lee Method: limit the day to a small ordered list and start with the first item.
- Warren Buffett style focus: the second-best priorities often become the real distractions.
- Peter Drucker: ask what contribution or output matters, not just what activity is available.
- OKR thinking: express work as observable outcomes, not vague intentions.
- Ray Dalio style principles: turn repeated lessons into reusable personal rules.
- Jeff Bezos one-way/two-way doors: move quickly on reversible choices and slow down for irreversible ones.
- Lean startup: when uncertain, run the smallest useful experiment.

## Identity Language

Use "AI solo operator" for the emerging worker who uses AI to operate like a small organization while still working independently.

Adjacent language:

- AI independent worker
- independent builder
- self-directed worker
- one-person company
- personal operating company
- work for ourselves

The skill is broader than entrepreneurship. It also fits creators, researchers, consultants, builders, freelancers, and people designing their own AI-assisted work system.

## Daily Questions

Good morning questions:

- If today only truly advances one thing, what should it be?
- What is the main contradiction blocking progress right now?
- Which task is real progress, and which task is just noise management?
- What worked yesterday that should be preserved?
- What failed yesterday that should be removed or redesigned?
- What is the smallest visible outcome that would make today count?
- What must not be touched today because it weakens the main battlefield?

## Planning Heuristics

- Prefer one finished asset over many started threads.
- Prefer a visible artifact over abstract effort.
- Prefer the root constraint over surface cleanup.
- Prefer a small proof over a large imagined system.
- Prefer a protected deep-work block over a perfect calendar.
- Prefer iteration over self-criticism.
